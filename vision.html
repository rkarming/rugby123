Future Vision

Accomplishments

This website was designed as a deliverable for Towson University's Fundamentals of Web Technologies and Development (AIT 616) class, instructed by Andrew Mangle, Ph.D., CISSP, CISA, PMP, CSSLP. The objectives of the course were as follows:

  1. Demonstrate proficiency in the use of HTML/XHTML, CSS, JavaScript, and PHP Web Technologies.
  2. Become familiar with dynamic HTML technologies.
  3. Understand current client and server side technologies.
  4. Understand web server interaction with backend databases.
  5. Design and publish websites.

Throughout the semester, students were assigned a total of five projects. Each project incorporated new material learned in class, with the overarching goal of satisfying all of the course objectives. Details about each project are listed below, along with the criteria required to score the highest grade.

Project 1: First Web Page

The purpose of this project was for students to create their first two web pages using HTML5. Students were instructed to create a Home page and a Contact page for an imaginary apiary called Charlee's Bee Company.

Project Objectives:

  1. Review HTML5 and its role in the development of the web.
  2. Build an HTML5 page including basic formatting and character elements.

Grading Criteria:

  • Three paragraphs of text (with examples) to explain the company and products, and to support the realism of the website
  • 15 or more HTML5 page elements (e.g. html, head, body, h#s, links, lists, special characters, form)
  • Two correctly functioning links (relative and absolute to web pages in separate directory folders)
  • Two appropriate images (with supported img element attributes) that are implemented correctly
  • Table of products that includes at least 12 rows/columns, thead, tfoot, and tbody
  • Contact form that includes labels, legend, and eight different input types
  • 12 semantic elements present
  • Ten structural elements and two different types of list elements present
  • Pages validate through W3C Markup Validator
  • Pages uploaded to the server with links provided in the Blackboard submission

Project 2: HTML and CSS

The purpose of this project was for students to expand their knowledge of Web Standards by utilizing the HTML experienced gained from Project 1 to add presentation elements to the structure of a website, along with an external CSS page. For this project, students were instructed to create an About page, an FAQ page, and a specific content page for a selected topic. The deliverables from this project served as the foundation for the remaining semester projects.

Project Objectives:

  1. Review HTML and its role in the development of the web.
  2. Combine the structure of HTML with the presentation abilities of CSS.
  3. Build HTML and CSS pages to support the Final Project.

Grading Criteria:

  • Three paragraphs (at least 400 words total, with no grammatical issues) per web page and at least five FAQs.
  • About page contains the site's goal, vision, 5 Ws and 1 H, and future use
  • More than two different colors and two different fonts used
  • At least five additional text formatting and structural elements present and used properly
  • At least five special characters, images, or graphics used properly to better explain the final project
  • Robust and effective use of HTML and CSS on all pages
  • Content, keywords, divs, meta tags, and structure are present and align with the topic
  • At least nine visual elements, graphics, and videos used (with at least one from each category)
  • Comments (at least eight), divs (at least 11), and paragraphs outline the structure and align with the content
  • Pages validate through W3C Markup Validator
  • Pages uploaded to the server with links provided in the Blackboard submission

Project 3: Layout

The purpose of this project was for students to add three web pages, each with a unique layout (Unsemantic, 960 Grid System, or a custom CSS framework), to the website created in Project 2. Students were instructed to create a Contact page (with a form), a Privacy Policy page, and a Links page (with a table), using a single external CSS file to style the pages.

Project Objectives:

  1. Understand the box model and CSS presentation elements.
  2. Apply the concepts of HTML as structure and CSS as presentation.
  3. Construct W3C compliant web pages.
  4. Create accessible and usable web page designs.

Grading Criteria:

  • Six or more different form elements (with a legend) present on the Contact page
  • Three templates (wireframes of the Contact, Privacy Policy, and Links pages using Unsemantic, 960.gs, and a custom CSS framework) are clearly legible, easily understandable, and utilized for the website, and are submitted in a single PDF file
  • At least 20 unique CSS rules (used to modify color, font style, font family, border, margin, and padding) present
  • More than five positioning approaches (e.g. static, relative, fixed, absolute, inline, block) present and used properly for the custom CSS framework
  • More than five positioning approaches (percentage based) present and used properly for the Unsemantic CSS framework
  • More than five positioning approaches (column based) present and used properly for the 960.gs CSS framework
  • Table, form, or privacy page properly and thoroughly formatted to aid understanding, using a single CSS page (does not apply to Unsemantic and 960.gs)
  • Links table contains a caption, th, and at least ten td & tr
  • Privacy policy page is complete and matches the site's actions
  • Pages validate through W3C Markup Validator
  • Pages uploaded to the server with links provided in the Blackboard submission

Project 4: JavaScript and Web Applications

The purpose of this project was for students to enhance their websites by adding JavaScript elements, along with some other common elements found in most modern websites. Students were instructed to set up a free Google Webmasters account, generate two sitemaps (.html and .xml), and create a robots.txt file. Additionally, students were to add three social media links (Facebook, Twitter, and Pinterest) to their sites; create and add a custom JavaScript game; and incorporate jQuery into a web page. Finally, students were expected to add form validation on the Contact page and a custom greeting on the Home page.

Project Objectives:

  1. Incorporate JavaScript into HTML including comments and alerts
  2. Manipulate the Document Object Model with JavaScript
  3. Change HTML and CSS attributes – modify both structure and presentation
  4. Understand and add validation to web-based forms

Grading Criteria:

  • Google Webmaster set up (with a screenshot to prove it), sitemap.html and sitemap.xml uploaded, and robots.txt in the project root directory
  • Three social media links (Facebook, Twitter, and Pinterest) present and appropriately linked
  • Form validation works with at least five different form elements (first name, email address (in the correct format), and three additional elements). When an error is present, CSS immediately shows the element's border in pink and provides valuable feedback to the right of the element
  • Custom greeting (i.e. "Good [morning, afternoon, evening], NAME") present on the Home page that uses local storage information from the Contact form
  • Custom JavaScript game works flawlessly, is related to the project topic, and is entertaining
  • jQuery component added. If taken from a jQuery script found on the Internet, it is modified to fit the student's website, the original source is cited in an HTML comment, and the script is explained in detail (at least five sentences) in the HTML comments
  • Pages validate through W3C Markup Validator
  • Pages uploaded to the server with links provided in the Blackboard submission

Project 5: Final Project

The purpose of this project was for students to apply knowledge gained throughout the course to deliver a complete client-side, web standards compliant website using HTML5, CSS, and JavaScript. Students were instructed to create additional web pages to meet the 10 page minimum for grading (12 page minimum for full credit).

Project Objectives:

  1. Combine all of the course material together into a complete website
  2. Apply the concepts of HTML as structure, CSS as presentation, and JavaScript as behavior
  3. Construct a W3C compliant website
  4. Create accessible and usable web pages
  5. Expand projects to build a complete website

Grading Criteria:

  • Custom JavaScript game converted from using popups/alerts to using HTML/CSS for user actions
  • Five additional specific content web pages added (at least 400 words per page) with wireframes submitted in a PDF
  • CSS for printing (print.css) created and considerably different and better than the default CSS
  • CSS for mobile devices (mobile.css) created and considerably different and better than the default CSS. Must validate through Mobile Emulator and Google's Mobile-Friendly Test
  • Content, presentation, and behavior of all pages are of excellent quality, usable, properly accessible, and support the project topic
  • Additional page outlining the future direction for the website added. Text outlines the semester's work, contains opportunities for new development, and is at least 250 words
  • W3C Markup Validator images on each page
  • All pages validate through W3C Markup Validator with required HTML tags

Development Opportunities

Although Start Playing Rugby Today! is a fully functioning client-side website, additional features could be incorporated to make it even better. Ideally, we would like our website to be the #1 reference for people learning about the sport of rugby union. Here are some development opportunities:

  • Add more features (e.g. keyboard controls, obstacle images of opponents, a background looped image of a pitch, game sounds) to the Try, Try, Try Again! game using JavaScript
  • Establish social media accounts for Rugby123 and link them to this website via the existing social media links (currently linked to Towson University's social media accounts)
  • Redesign the Home page using CSS and JavaScript to make it more exciting
  • Start Playing Rugby Today! is a mobile-friendly site. However, it could be built into a mobile app
  • Add an Equipment page to provide beginners with information about products such as rugby boots
  • Develop the server side of the website
